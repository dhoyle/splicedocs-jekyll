Show Command

The show command is used to display information about active connections and database objects. You can use a number of different forms of this command to show different types of information, as shown in the following syntax diagram:

Syntax

SHOW
{
   CONNECTIONS                                 | 
   FUNCTIONS [ IN schemaName ]                 |
   INDEXES [ IN schemaName | FROM table-Name ] |
   PRIMARYKEYS FROM schemaName.FROM tableName  |
   PROCEDURES [ IN schemaName ]                |
   ROLES                                       |
   SCHEMAS                                     |
   SYNONYMS [ IN schemaName ]                  |
   TABLES [ IN schemaName ]                    |
   VIEWS [ IN schemaName ]                     |
}

 

Each form of the SHOW command has its own topic page:

Command Description Usage
show connections Displays a list of connection names and URLs. splice> show connections;
show functions Displays the functions in a database or in a specific schema. splice> show functions;
show indexes Displays the indexes in a database, a schema, or a table. splice> show indexes;
show primarykeys Displays the primary keys in a table. splice> show primarykeys from myschema.myTbl;
show procedures Displays the procedures that have been created in the database. splice> show procedures;
show roles Displays a sorted list of all of the roles defined in the database. splice> show roles;
show schemas Displays all of the schemas in the current connection. splice> show schemas;
show synonyms Displays the synonyms that have been created in a database or schema. splice> show synonyms;
show tables Displays all of the tables in a database or schema. splice> show tables;
show views Displays all of the views in a schema. splice> show views;
