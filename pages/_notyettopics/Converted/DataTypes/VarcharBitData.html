VARCHAR FOR BIT DATA

The VARCHAR FOR BIT DATA data type allows you to store binary strings less than or equal to a specified length. It is useful for unstructured data where character strings are not appropriate (e.g., images).

Syntax

{ | CHAR VARYING | CHARACTER VARYING } (length) FOR BIT DATA

length

An unsigned integer literal designating the length in bytes. There is no default length for this type. The maximum size of the length value is 32,670 bytes.

JDBC Metadata Type (java.sql.Types)

VARBINARY

Example

VARCHAR (2048) FOR BIT DATA;

Usage

Here are several usage notes for the VARCHAR FOR BIT DATA type:

  • VARCHAR FOR BIT DATA stores variable-length byte strings; values are not padded out to the target length.
  • An operation on a VARCHAR FOR BIT DATA and a CHAR FOR BIT DATA value (e.g., a concatenation) yields a VARCHAR FOR BIT DATA value.
  • The type of a byte literal is always a VARCHAR FOR BIT DATA, not a CHAR FOR BIT DATA.
