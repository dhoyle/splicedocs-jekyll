SYSCS_UTIL.SYSCS_KILL_STALE_TRANSACTIONS

The SYSCS_UTIL.SYSCS_KILL_STALE_TRANSACTIONS system procedure kills all active transactions up to the specified killable maximum transaction ID.

This is used to kill stale transactions, usually to allow DDL operations to execute.

Syntax

SYSCS_UTIL.SYSCS_KILL_STALE_TRANSACTIONS(maxTransactionId)

maxTransactionId

A long integer that specifies the transaction ID of the maximum transaction ID that can be killed.

Results

This procedure does not return a result.

Example

splice> create index aidx on a(i);
ERROR SE001: Splice Engine exception: unexpected exception
ERROR XJ001: Java exceptions; 'There are active transactions [51, 52]:
   java.lang.RuntimeException'.

splice> call SYSCS_UTIL.SYSCS_KILL_STALE_TRANSACTIONS(52);


Statement executed.
