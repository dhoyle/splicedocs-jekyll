SYSCS_UTIL.VACUUM_SCHEMA

The SYSCS_UTIL.VACUUM_SCHEMA system procedure improves the performance of your database by cleaning no longer needed historical update data from a schema.

Splice Machine recommends calling this system procedure regularly (or frequently) on schemas in your database that are frequently updated with modifications or deletions. Insertions by themselves do not generate the kind of data that needs to be vacuumed from a schema; however, if a row is frequently inserted, then deleted, then inserted again, and that row has a primary key or indexed fields, vacuuming will be required.

Many customers use a scheduling utility such as Cron to call this procedure.

Syntax

SYSCS_UTIL.VACUUM_SCHEMA( VARCHAR(128) schemaName );

schemaName

Specifies the schema to be vacuumed. Passing a null or non-existent schema name generates an error.

Results

This procedure returns a table with the following columns:

Column Description
CONGLOMERATEID The number of the conglomerate that was vacuumed.
NUMPARTITIONS The number of partitions (regions) that were vacuumed.
TIMETAKEN_S The total time used to vacuum this conglomerate.
ROWS_READ The total number of rows read during the vacuum.
ROWS_VACUUMED The total number of rows that were vacuumed.
ROWS_DELETED The number of vestigal rows that were deleted during the vacuuming.

SQL Examples

splice> CALL SYSCS_UTIL.VACUUM_SCHEMA( 'MySchema');
Statement executed.
